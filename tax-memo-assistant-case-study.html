Problem & Solution

Why a plain AI chatbot isn't safe for tax memos

Tax associates at professional services firms lose real time formatting first-draft memos — but handing this to an unguarded AI creates a bigger risk than the time it saves.

The problem

An unguarded AI can invent legal citations, invent client facts that were never given, and state confident conclusions on questions it was never actually equipped to answer — all while sounding completely credible. In a regulated professional context, that's a liability, not a shortcut.

The solution

A governed Claude Skill that enforces a fixed six-section output, refuses to use any fact or citation the user didn't supply, always hedges its conclusions, and always attaches a disclaimer — turning a risky shortcut into a safe drafting accelerator.

Process

How it was built, step by step

The build followed the same process a real AI Center of Excellence team uses to ship a governed internal tool.

01

Write the Architectural Standard

Defined the fixed six-section output format, tone rules, and quality bar before writing a single instruction for the AI itself.

02

Define the guardrails

Set seven non-negotiable rules: no invented citations, no invented facts, no definitive conclusions, mandatory disclaimer, no client-ready polish, no jurisdiction assumptions, escalate over guessing.

03

Build the Claude Skill

Translated the standard into an actual Skill file Claude auto-applies — including trigger conditions and a pre-delivery quality checklist.

04

Build a tiered prompt library

Wrote 15 test scenarios across three tiers — complete facts, missing facts, and ambiguous or multi-issue authority — then ran 10 of them in this QA pass.

05

Run structured QA

Executed 10 real test runs, scoring each against a fixed rubric — hallucination risk, missing-fact flagging, disclaimer presence, and conclusion hedging.

06

Diagnose and confirm fixes

Two runs failed the same way. Rerunning them with manual invocation resolved both — proving the failure was in trigger detection, not the skill's core logic.

Output Format

Every memo follows the same fixed structure

No section is ever skipped, merged, or reordered — regardless of how the request is phrased.

01IssueA precise, neutral statement of the tax question presented
02FactsOnly facts the user supplied — every gap explicitly flagged
03Applicable LawOnly authorities the user named — never invented
04AnalysisHedged, exploratory reasoning applying law to facts
05ConclusionA preliminary view framed as a starting point, never final
06DisclaimerMandatory on every output, with zero exceptions
Guardrails

Seven non-negotiable rules

These rules exist specifically to prevent hallucination — an AI stating something false or unsupported as if it were verified fact.

No invented citations
No invented facts
No definitive conclusions
Mandatory disclaimer, every time
No client-ready polish
No jurisdiction or entity assumptions
Escalation over guessing

# Tax Memo Assistant

## Purpose

Produce a first-draft tax research memo for a tax associate at a professional
services firm, based only on the facts, issues, and authorities the user
actually supplies. This is a drafting accelerator, not an opinion-generator.

## When to use this skill

Trigger on:
- Direct requests for a tax research memo or "memo on [issue]"
- A user describing a tax fact pattern and asking "how should I write this up"
- Requests to revise, shorten, expand, or reformat an existing memo

## Fixed output structure

Every memo must contain all six sections below, in this order, never omitted,
merged, or reordered — even if the user asks for a shortened version.

1. **Issue** — a precise, neutral statement of the tax question(s) presented.
2. **Facts** — only facts the user supplied, stated plainly.
3. **Applicable Law** — only authorities the user supplied, or authorities

Known Limitations

Documented honestly, not hidden

A trustworthy QA process reports what broke and why — not just a pass rate.

Auto-trigger detection is unreliable for direct lookup questions

The skill reliably activates for explicit requests ("write this up," "draft a memo") but not for phrasing like "does this qualify for X credit?" In both confirmed failures, when the skill didn't trigger, Claude either recalled unverified training knowledge or performed a live web search and cited named third-party sources — both violating the no-invented-citation rule. Manually invoking the skill resolved the issue in both cases.

A gray zone around procedural mechanics

The skill sometimes explains well-known procedural steps (like standard election forms) without a citation, treating them as general professional knowledge. This is a deliberate judgment call worth reviewing, not a clear violation.

Relevance
